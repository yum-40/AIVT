# 今後やりたいこと

AIVTの今後の実装方針メモ。Claude Codeにも見せる前提で、やる順に並べる。

## 作業順序

1. MVPの診断フローを作る
2. 回答途中の保存と復元を入れる
3. 結果表示を作る
4. SNS共有しやすい結果カードを作る
5. CSV / JSON出力を作る
6. Supabaseを入れる
7. 同意がある場合のみ診断結果を保存する
8. 開発者用Google認証を入れる
9. 管理画面から問題・選択肢を編集できるようにする
10. 問題セットと診断ロジックのバージョン管理を入れる
11. アップデート履歴ページを作る
12. UIとコード側の両方から更新履歴を登録できるようにする
13. グラフ・タグ・日別揺らぎ比較を追加する
14. 英語版を作る
15. 応募や公開に向けてREADME・デモ・利用実績を整える

## 重要メモ

- 最初から全部作らない。まずは壊れにくいMVPを優先する。
- 問題数が多くなるため、ブラウザバックやリロードで回答が消えない設計にする。
- MVPではlocalStorageで途中保存し、あとでDB保存を追加する。
- 診断結果には、モデル名、ペルソナ名、診断日時、タイムゾーン、問題セットバージョンを残せるようにする。
- 位置情報は扱わず、時刻とタイムゾーンだけを使う。
- SNS共有用の楽しい表示と、CSV / JSONなどのデータ出力を両方用意する。
- 英語版も将来的に出す。

## バージョン方針

`1.0.2` のような形式にする。

- 3つ目の数字：誤字修正、文言調整など軽微な更新
- 2つ目の数字：問題追加、UI改善、診断ロジック調整など中規模更新
- 1つ目の数字：大幅アップデート、診断設計の大きな変更

## Claude Codeへの注意

Claude Codeは、まずこのドキュメントを読んだ上で、いきなり全部作ろうとしないこと。

大きな変更をするときは、必ず以下を出す。

- 変更予定ファイル
- 実装方針
- DB設計への影響
- 途中保存への影響
- セキュリティ懸念

実装後は、変更ファイル、実行コマンド、lint / build / test結果、未確認事項を報告すること。
